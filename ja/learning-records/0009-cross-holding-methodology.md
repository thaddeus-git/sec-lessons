# Learning Record 0009 — クロスホールディング分析の方法論

**Date**: 2026-06-17
**Lesson**: 0009-cross-holding-methodology.html
**Status**: 完了

## 重要な洞察

1. **各ページが1つの具体的な投資質問に答える**ことがクロスホールディングレポートの中心的な設計哲学である。IHS P4レポート10ページ = 10の質問。データの羅列 ≠ レポート。
2. **ページを跨いだクロス検証こそが真の洞察である**。4つの古典的なシナリオ：
   - 確認型（P2 大口保有 + P3 買い増し + P5 Top Buyer）= 強いシグナル
   - 矛盾型（P2 大口保有 + P3 小幅減らし + P6 非Top Seller）= リバランスの可能性
   - 危険型（P4 アクティビスト + P7 新規保有）= 高度な警告
   - 転換型（P8 パッシブ資金増加 + P9 バリュー／グロースのローテーション）= 業界ライフサイクルの変化
3. **正しいレポートの読む順序**: 30秒の概要把握 → 2分の傾向把握 → 5分の深掘り → 15分の調査。Dashboardはこのリズムをサポートする設計とする（概要カードを最も目立つ位置に、詳細データはタブ内に格納）。
4. **6つの設計原則**：
   - 各ページが1つの質問に答える
   - 各ページ下部にCommentary空欄を設ける（アナリストが判断を記入するため）
   - 表の列は業界標準に揃える（Style / Turnover / Peer Averageは必須）
   - ランキング表には必ずランキング指標を明記する
   - グラフと表はセットで用いる（円グラフで分布、棒グラフでフローを示す）
   - 免責事項は可視化する（各ページ下部、文書末尾ではない）
5. **Top Buyers/Sellersが一方向のみ = 99%はバグ**。通常は双方向（買う人もいれば売る人もいる）。よくある原因：期間比較の誤り、フィールドマッピングの誤り、データソースの欠落。**セルフチェックリストに「BuyersとSellersの両方にデータがあるか」を必ず含める**。
6. **期間比較は時価総額（value delta）ではなく株数（shares delta）で見る**。時価総額の変化は株価変動に影響される。Top Buyersは「能動的な買い増し」または「株数の変化」でソートする必要がある。
7. **Carvana 2026Q1実践分析**: Vanguard/BlackRockの保有は不変（パッシブインデックス資金は安定）+ Baupostが$480Mを新規取得（バリュー投資家の再参入）+ アクティビストなし = 「インデックス安定＋バリュー再評価＋アクティビスト干渉なし」→ ややポジティブな状態。
8. **出力品質の10項目セルフチェックリスト**: 単位、ランキング指標、institution_name、delta_pctの境界値、Buyers/Sellersの対称性、隣接期間比較、免責事項、タイムラグ、Item 4レビュー、中国ADRの「適用外」表示。
