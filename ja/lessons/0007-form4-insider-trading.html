Form 3/4/5 とインサイダー取引
経営陣が投票で示すシグナルの読み方

Lesson 0006 では外部機関（Vanguard、BlackRock など）が何を買い、何を売っているかを学びました。このレッスンでは会社の内部者（CEO、CFO、取締役）自身が何を買い、何を売っているかを扱います。

インサイダー取引データは SEC 開示制度の中でシグナル対ノイズ比が最も高いデータの一つです——内部者は外部投資家よりも会社に関する理解がはるかに深いため、彼らの買いや売りは通常、機関保有の変動よりも情報量が豊かです。ただし、その前提として正しく読み解くことが必要であり、そうしなければ誤った判断や誤解を招く恐れがあります。

インサイダー取引は「イベント」ではなく、継続的なデータフローです。単発の取引は意味を持ちませんが、加重された総合的なセンチメント（InsiderSentiment 指標）は、「この会社の経営陣は将来についてどう考えているか」を教えてくれます。

Section 16 とは何か、誰を対象とするか

Section 16 は 1934 年証券取引法の第16条で、会社内部者の株式保有と取引の開示を規定しています。3つの重要な境界があります。

対象者意味具体例
Officer（役員）会社の上級管理職、通常は VP 以上CEO、CFO、COO、各 SVP/EVP
Director（取締役）取締役会メンバー社外取締役、親会社から派遣された取締役
10% Owner（10% 以上株主）会社の >10% の株式を保有する株主創業者、親会社、大型パッシブ株主

⚠️ 中国 ADR 企業の盲点

Section 16 は米国登録証券を保有する内部者のみに適用されます。中国 ADR 企業（Uxin、Autohome）の経営陣は通常、Cayman/VIE レベルの权益を保有しています——これらの权益は SEC Section 16 の管轄外です。

結果として、UXIN/ATHM には Form 4 データがほぼ存在しません。これは「インサイダー取引がない」ということではなく、「インサイダー取引が SEC 開示の範囲外である」ことを意味します。当システムでは config.py で has_section16 = False とマークし、Dashboard には「データなし」ではなく「該当なし」と表示します。

3種類のインサイダー開示フォームの違い

インサイダー取引の開示には3つのフォームがあり、提出のタイミングと内容が異なります。簡単に区別すると以下の通りです。

フォーム正式名称いつ提出するか内容
Form 3Initial Statement内部者となってから10日以内初期保有状況の申告（1回限り）
Form 4Statement of Changes取引後2営業日以内売買の都度報告
Form 5Annual Statement会計年度終了後45日以内年次集計（除外取引）

Form 4 が最も重要——タイムリー性が最も高く（2営業日）、全ての取引をカバーします。Form 3 と Form 5 は主に「事後的補足」の性質を持ちます。

Form 4 の構造：Table I vs Table II

Form 4 は2つのテーブルから構成されており、その違いは極めて重要です。 

Table I — Non-Derivative Securities（直接保有株式の売買）
  フィールド:
    - Title of Security          銘柄名
    - Transaction Date           取引日
    - Transaction Code           P=買付, S=売却, G=贈与, A=付与, F=税金支払いのための売却
    - Amount                     株数
    - Price                      単価
    - Amount After               取引後保有株数

Table II — Derivative Securities（オプション/デリバティブの変動）
  フィールドは類似しているが、単位は「株式数」ではなく「オプション数」
  よくある Code: M=権利行使, A=付与（オプション）, C=転換

よくある取引コード（Transaction Code）

Code意味シグナルはあるか？
P能動的買付 (Purchase)✅✅✅ 強いポジティブシグナル——内部者が自腹を切っている
S能動的売却 (Sale)✅ 弱いネガティブシグナル——分散保有の可能性もあり、必ずしも弱気とは限らない
F税金支払いのための売却 (Tax withholding)❌ ほぼシグナルなし——オプション権利行使時の源泉徴収は標準的な処理
G贈与 (Gift)❌ 弱いシグナル——慈善・家族信託でよく見られる
A付与/権利行使 (Award/Exercise)❌ シグナルなし——報酬の一部であり、能動的な取引ではない
Mオプション権利行使権利行使後の売却行動（Code F/S の組み合わせ）で判断

最もよくある誤判定

「Code F の大量の税金支払い売却」を「内部者の弱気」と誤読すること。これが最大の落とし穴です——経営陣が 10b5-1 計画のもとでオプションを一括権利行使する際、大量の Code F 取引（源泉徴収）と Code S 取引（売却）が同時に発生します。

正しい方法は、Code P（能動的買付）の金額と、Code S から権利行使に伴う売却を差し引いた純売却額を見ることです。単純に全ての売却コードを SUM すると、シグナルがノイズに埋もれてしまいます。

インサイダーセンチメント指標の正しいアルゴリズム

単純に売買の件数を数えても意味がありません（10b5-1 計画は大量のペア取引を生み出します）。正しい方法は加重スコアリングです。 

InsiderSentiment = Σ(買付金額_i × 役割重み_i × 取引コード重み_i)
                  - Σ(売却金額_j × 役割重み_j × 取引コード重み_j)

役割重み:
  CEO / CFO                   2.0    最高経営層
  10% Owner                   1.5    大株主
  Director                    1.0    取締役会
  EVP / SVP / VP              0.8    執行役員

取引コード重み:
  P (能動的買付)              1.0
  S (能動的売却)              1.0
  F (税金支払い売却)          0.2
  G (贈与)                    0.3
  A (オプション付与)          0.0
  M (オプション権利行使)      0.0   (権利行使後の売却のみを対象とする)

[-100, +100] に正規化:
  > +30  → 強気シグナル
  -30 ~ +30 → 中立
  < -30  → 弱気シグナル

F の重みを 0 ではなく 0.2 としている点に注意——わずかなシグナルを残すためです（大量の税金支払い売却は大規模な権利行使＝集中換金を反映する可能性がありますが、結論を支配させるわけにはいきません）。

よくある間違いを避けるために：10-K の発表1ヶ月前には、大量のインサイダー売却（10b5-1 計画で事前設定されたもの）が発生します。これは必ずしも「経営陣が有価証券報告書に自信がない」ことを意味しません——計画に従って 10b5-1 フレームワークのもとで分散保有している可能性があります。重要なのは 10b5-1 計画の提出日（通常 8-K の Item 8.01 または DEF 14A に記載）であって、取引そのものではありません。

Form 144 — 売却の「予告編」

Form 144 は別のタイプのインサイダーシグナルです——事前通知であり、事後開示ではありません。

Form 144 の重要事項
発動条件：内部者が ≥5,000 株 または ≥$50,000 の売却を計画
提出タイミング：売却予定の 1～3 営業日前
有効期間：計画売却は Form 144 提出後 90 日以内に完了する必要あり
aggregate market value：提出日の株価に基づく見積額（実際の約定価格ではない）

重要な解釈：

  • Form 144 は「すでに売却した」こととイコールではない——あくまで「売却の準備」です。計画が実行されないこともあります
  • しかし、複数の Form 144 が継続的に提出される = 継続的な売却シグナルであり、警戒が必要です
  • Form 144 で売却圧力を予測する際は、「実際の執行率」（OpenInsider サイトにこのデータがあります）を確認します

中国競合の特別な扱い（実務ルール）

当システムの5社の競合のうち、Uxin (UXIN) と Autohome (ATHM) は中国 ADR です。これらの企業には Form 4 / Form 144 データはありませんが、親会社レベルの香港株・A株開示は存在します。

システム処理ルール

1. config.py でのマーク：has_section16 = False
2. Dashboard 表示：
    - 「データなし」ではなく「該当なし」（取得失敗と誤解されるのを防ぐ）
    - 横に注釈：「内部者は VIE レベルの权益を保有しており、SEC 開示の対象外」
3. 異常アラート：
    - 中国 ADR に突如 Form 4 が出現した場合 → 赤色警告
    - これは稀なイベントであり、内部者が意図的に权益を Cayman から米国レベルに移して取引していることを示す
    - 歴史的にこのケースは極めて稀だが、発生した場合ほぼ常に大きな出来事が起きている

Carvana の直近5ヶ月のインサイダー取引

下表は 2026年1月～6月に EDGAR から取得した Carvana Form 4 データ（簡略版）です。

日付内部者役割Code株数金額 ($K)加重後
2026-06-11Ernest C. Garcia III10% Owner (2.0)S50,00015,000-30,000
2026-06-11Mark JenkinsCFO (2.0)F8,0002,400-960
2026-05-22Paul KeisterVP (0.8)P2,000600+480

これら4件の取引の InsiderSentiment = (-30,000 - 960 + 480 - 90,000) / 60,000 = -200% → 正規化後 約 -100（極度の弱気）。

ただし注意点：Ernest C. Garcia III は 10% Owner であり、彼の大口売却は多くの場合、10b5-1 計画に基づく能動的な分散保有であり、事業への弱気を示すものではありません。彼が 8-K または DEF 14A で開示した 10b5-1 計画を確認して初めて、性質を判断できます。

Quiz

Q1: 経営陣が 10b5-1 計画のもとでオプションを一括権利行使し、100件の Code F（税金支払い売却）取引が発生しました。あなたのシステムはどう解釈すべきですか？

クリックして答えを見る

ほぼネガティブシグナルとは解釈しません——これは 10b5-1 計画の標準的な処理です。Code F の重みは 0.2 であり、100件 × 0.2 = 20 単位のノイズ寄与に過ぎません。件数だけで見ると「インサイダーが大規模に売却している」という誤った結論に至りますが、加重アルゴリズムを使えば、結論は中立かやや弱気になります。判断時には、当該 10b5-1 計画がいつ提出されたか（通常 8-K Item 8.01 または DEF 14A に記載）を確認する必要があります。提出が 3～6ヶ月前であれば、基本的に計画に沿った行動です。

Q2: Uxin (UXIN) の Dashboard に「インサイダー取引：該当なし」と表示されています。これはデータ取得失敗とどう区別しますか？

クリックして答えを見る

両者は明確に区別する必要があります。意味が全く異なるからです。「該当なし」= 会社の構造上、Section 16 開示が適用されない（中国 ADR の内部者は VIE レベルの权益を保有）、システムは config.py で has_section16 = False とマークしています。「データなし」= 取得失敗、またはその期間に Form 4 が存在しない（Form 4 は2営業日以内の強制開示であるため稀）。Dashboard の UI デザインでは、異なる色と文言を使うべきです（「該当なし」は灰色、理由を注釈；「データなし」は赤色、API 障害の可能性を示唆）。

Q3: Carvana の CFO が 10b5-1 計画のもとで $5M の株式を売却し、同時に VP が能動的に $1M を買い付けました。あなたのシステムはどうスコアリングしますか？

クリックして答えを見る

この $5M 売却の具体的な Code によります。Code F（税金支払い売却）+ 付随する Code S（10b5-1 計画下の売却）の場合、重み = 0.2 + 1.0 × 役割重み 2.0 = 2.2 単位のネガティブ寄与となります。一方、VP の能動的買付 (Code P) = 1.0 × 0.8 = 0.8 単位のポジティブ寄与です。金額差が 5:1 の場合、結果は純ネガティブで約 -3,400 単位となりますが、正規化後は -30 ～ 0 の範囲に収まる可能性が高く、結論は「慎重」であり、「弱気」ではありません。つまり金額が大きくても、10b5-1 計画に基づくプログラム売却であるため、結論は割り引いて評価すべきです。
