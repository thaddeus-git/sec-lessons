13Fと機関保有
クロスホールディング分析のデータ基盤

これまでの5つのレッスンでは企業自身が開示する情報——10-K、10-Q、8-K、Earnings Call を企業目線で見てきました。このレッスンからは第三者による企業への見方——誰が買い、誰が売り、誰が大量保有し、誰が撤退しているかを扱います。これは IHS Markit レポート（P4 クロスホールディング分析）のコアデータソースです。

13F は米国 SEC が大手機関投資家に義務付ける保有状況報告書で、正式名称は Schedule 13F です。公開、無料、四半期ごとに更新されるため、クロスホールディング分析の事実上の標準となっています。

IHS Markit の参考レポート P2-P7 はほぼ全て 13F データに基づいています。当システムも 13F を使用しています——ただしデータソースを IHS から SEC EDGAR に切り替えています。

13Fとは

13F を「機関投資家の四半期成績表」と考えてください——ただし知っておくべきいくつかのハードルールがあります。

ルール具体要件戦略的意味
法的根拠Securities Exchange Act of 1934, Section 13(f)法定開示、提出しないと違法
報告基準運用資産 > $1億の機関投資家ほぼ全ての大手ミューチュアルファンド、ヘッジファンド、年金基金、ETF発行体をカバー
報告頻度四半期終了後45日以内Q1 締切 5/15、Q2 締切 8/14、Q3 締切 11/14、Q4 締切 2/14
報告機関数約5,000～6,000米国市場の「スマートマネー」をほぼカバー
13F が示すのは四半期末の保有スナップショット——四半期内の取引ではありません。ある銘柄が3月に買われ、4月に売られても、13F には全く現れません。IHS Markit レポートの Turnover 列は、隣接する2期の 13F から推計した「ポジション調整の度合い」であり、実際の取引フローではありません。

13Fが開示するものとしないもの

13F の「境界」を理解することは、その内容を知ること以上に重要です——これらの境界が、どのような質問に答えられ、どれに答えられないかを直接決定するからです。

開示される内容（13Fに含まれるもの）

  • 各保有銘柄：名称、CUSIP、株数、時価総額
  • 時価総額フィールドは value_x1000、単位は千米ドル（ドルでも百万ドルでもありません）
  • 全ての米国登録証券のロングポジションをカバー

開示されない内容（13Fの盲点）

見えないものなぜ見えないか分析への影響
ショートポジション13F はロングのみ報告ヘッジファンドの空売り賭けが完全に欠落——これが最も強いシグナルである可能性も
オプション/デリバティブ13F はデリバティブを含まない多くのヘッジファンドはオプションで見解を表明するが、13F では確認不可
米国以外の保有13F は米国株のみ世界の機関が非米国株で持つポジションは全く見えない
非上場証券私募債、未上場企業は 13F の対象外PEファンド、private credit ファンドの活動は見えない
四半期内の売買期末のスナップショットのみ表示日内、週内、月内の取引はすべて「期末保有」に平均化される
報告機関の $1億未満の部分基準未満は報告不要小規模機関や個人投資家の行動は 13F の対象外
覚えておくべき一句话：13F が見せるのは「期末の一枚の写真」であり、「四半期の動画」ではありません。動画を見るには、より詳細なデータ（Bloomberg の取引フロー、WhaleWisdom の有料サブスクリプションなど）が必要です。

単位変換の落とし穴（バグ多発ゾーン）

13F データ処理で最も間違いやすいのは単位変換です。以下の表は実際のバグパターンです。

SEC 原始 XML: <value>12345</value>  →  意味: $12,345,000  →  $12.345M
edgartools 返却: フィールド名 value_x1000 → 数値 12345 → 意味: 12345 × $1,000

よくある間違い:
  間違い A: value_x1000 を「千ドル」として表示 → 数値が 12,345 に見える → 実際は $12.345M
            1000倍のズレ
  間違い B: 12345 をさらに 1000 で割って 12.345 と表示 → 1,000,000倍のズレ
  間違い C: 集計時に時価総額の規模を区別しない
            → Vanguard の CVNA 保有 $1.2B（value_x1000 = 1,200,000）
            → BlackRock の UXIN 保有 $3M（value_x1000 = 3,000）
            → 単純に SUM すると UXIN の変動が CVNA に埋もれる（差は400倍）

検証方法:
  1. Vanguard の CVNA に関する 13F の元 XML を取得
  2. <value> フィールドを手動で読み取る = X
  3. edgartools で解析後の value_x1000 と比較
  4. CVNA の当時の株価 × 株数でクロス検証（概ね一致、±5%の誤差を許容）

⚠️ 当システムの対応方法

データフロー全体で value_x1000 フィールドは SEC の元の単位（千ドル）を維持し、dashboard 表示時のみ一律に 1000 で割って百万ドル（$M）で表示します。 内部計算（集計、QoQ 変動、Top Buyers/Sellers 順位）は全て統一された「$M」単位で行い、データフローの異なる箇所で単位を混在させないようにします。

13F の期間比較の正しい方法

クロスホールディング分析の中核は「隣接する2期の 13F を比較すること」です。このロジックを正しく実装しないと、Top Buyers/Sellers の順位が完全に歪みます。

境界条件（必ず処理すべき4つのケース）

  1. 新規保有：curr_value > 0 かつ prev_value == 0 — ある機関が初めて出現
  2. 清算：curr_value == 0 かつ prev_value > 0 — ある機関が完全に撤退
  3. 増加：curr_value > prev_value — 保有増
  4. 減少：curr_value < prev_value — 保有減
  5. 不変：curr_value == prev_value — 変動なし（稀、株数不変かつ株価不変の場合のみ）
curr = holdings WHERE report_period = '2026-03-31'
prev = holdings WHERE report_period = '2025-12-31'
merged = curr FULL OUTER JOIN prev ON (institution_cik, ticker)

for each row:
    if curr_value > 0 and prev_value == 0:  → Initiation
    if curr_value == 0 and prev_value > 0:  → Liquidation
    if curr_value > prev_value:              → Increase
    if curr_value < prev_value:              → Decrease
    if curr_value == prev_value:             → No change

注意:
  - prev_value = 0 の場合は delta_pct を計算しない（ゼロ除算）
  - FULL OUTER JOIN 後、institution_name がいずれかの側で NULL になる可能性があるため補完が必要
  - 期間比較は必ず隣接する2期で行う（飛ばし期間は不可）、さもなければ変動が人為的に拡大される

銘柄ごとの変動 = 能動的な売買 + 株価変動

ある機関が CVNA を 10,000 株保有し、今期の 13F でも 10,000 株だが、株価が $100 から $150 に上昇した場合：

  • 株数は変わっていない = 当該機関は能動的に売買していない
  • しかし時価総額は $1,000,000 から $1,500,000 に増加 = 純粋に株価上昇によるもの

この「受動的な時価総額の上昇」は誤って「買い増し」と分類され、Top Buyers ランキングを誤導します。正しい方法は、時価総額の変化（value_change）ではなく、株数の変化（shares_change）を見ることです。

中核原則：13F の期間比較では、能動的な売買の判断に株数（shares）で見る、時価総額（value）で見ない。時価総額の変化は株価変動で汚染されます。株数の変化だけが真の「買い/売り」を示します。

クロスホールディング分析における13Fの役割

当モニタリングシステムシリーズの最終成果物（IHS スタイルレポート）の入出力対応関係：

入力出力13Fで回答可能か？
25のシード機関 × 5競合 × 2期の13Fクロスホールディング行列 (機関 × 競合)✅ 完全
QoQ 保有変動 (delta value / delta shares)✅ 完全（shares ベースがより正確）
競合間の Buyers/Sellers ランキング✅ 完全
Initiations & Liquidations✅ 完全
Churn Proxy (turnover 近似)⚠️ 近似（実際の turnover ではない）
投資スタイル分類 (Value/Growth/GARP)❌ 不可（全保有 + ファンダメンタルデータベースが必要）

13F は「誰が売買しているか」に答えることはできますが、「なぜ売買するのか」には答えられません——後者は経営陣の commentary を見る必要があります（13F は取引理由を開示しません）。だからこそ、13F の変動を 8-K や Earnings Call とクロス検証する必要があるのです。

Carvana での具体例

下表は 2026Q1（report_period 2026-03-31）に EDGAR から取得した Carvana (CVNA) の Top 5 機関保有者（value 順）：

機関株数 (千株)value_x1000 ($K)換算 $M
Vanguard Group12,8001,920,000$1,920M
BlackRock10,2001,530,000$1,530M
State Street5,400810,000$810M
Baupost Group3,200480,000$480M
D1 Capital2,800420,000$420M

単位変換の検証：value_x1000 = 1,920,000 → 実際の金額 = 1,920,000 × $1,000 = $1,920,000,000 = $1,920M = $1.92B。✓

Vanguard の $1.92B 保有は、UXIN（時価総額が一桁小さい）の全機関保有合計の数十倍に相当します。これが、集計時には機関ごとに個別ランキングしなければならず、単純に SUM してはいけない理由です——さもなければ UXIN のシグナルが Carvana に完全に埋もれてしまいます。

Quiz

Q1: 13F 報告書の value_x1000 フィールドの値が 12345 の場合、これは何ドルを表しますか？

クリックして答えを見る

12,345,000 ドル（$12.345M）

value_x1000 のセマンティクスは「単位は千ドル」です。したがって値 12345 = 12345 × $1,000 = $12,345,000 ≈ $12.3M となります。よくある間違いは、"$12,345" とそのまま表示するか、逆に 1000 で割って "$12.345" と表示することです。それぞれ 1000倍、1,000,000倍のズレが生じます。

Q2: ある機関の CVNA 株数が今期も 100 万株のまま、時価総額が $1億から $1.5億に増加しました。これは「買い増し」とみなせますか？

クリックして答えを見る

買い増しではなく、受動的な時価総額の上昇です。株数が変わっていない = 能動的な売買はありません。時価総額の変動は CVNA の株価が $100 から $150 に上昇したことに完全に起因します。Top Buyers ランキングでは、株数の変化（shares delta）でソートすべきであり、時価総額の変化ではありません。そうしないと、「保有したまま＋株価上昇」を「能動的な買い増し」と誤判定することになります。

Q3: 13F から「ヘッジファンドの CVNA におけるショートポジション」を知ることはできますか？

クリックして答えを見る

できません。13F はロングポジション（long positions）のみ開示し、ショートポジションやオプションは開示しません。ショートポジションを確認するには別のデータソース（S3 Partners、Markit Short Interest、Finra 隔週ショートレポートなど）が必要です。これこそ IHS Markit レポートが "data limitations" と明記し、読者に 13F の盲点を注意喚起している理由です。
