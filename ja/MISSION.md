# ミッション

## なぜこれを学ぶのか？

私は中国の中古車取引会社に所属しており、上場準備を進めています。米国市場で同じセクターに属する上場企業の動向を監視し、戦略投資の参考とする必要があります——競合の財務状況、戦略的行動、リスクシグナルを理解することで、自社の上場タイミングとビジネス戦略を調整するためです。

この**監視システムの背後にあるドメイン知識**（コーディング実装ではなく）を習得する必要があります。その習得レベルは、戦略投資部の同僚に対し、このシステムの各構成要素が何をするものか、なぜそのように設計されているか、見えた結果が何を意味するかを説明できる程度です。

## 何ができるようになるのか？

- SEC filing の種類、構造、重要情報を読み解き、どこに何があるかを把握できる
- 中古車セクター競合モニタリングの中核的ロジックを理解する：誰を監視するか、どの指標を見るか、なぜそうするか
- システムが出力するダッシュボードやサマリーを見て、「この情報は重要か、上層部に報告すべきか」を判断できる
- 戦略投資部の内部会議で、この監視システムの動作方法とアウトプットの価値を説明できる

## 学ばないこと

- Python プログラミング、Streamlit 開発、API 呼び出し — すでに習得済み
- LLM prompt エンジニアリング — coding agent に任せる
- SEC の法律コンプライアンス詳細 — これは法律の授業ではない

## 成功基準

- SEC に詳しくない同僚に対して、10-K、10-Q、8-K がそれぞれ何であり、なぜすべてが重要なのかを自分で説明できる
- 選んだ5社の競合とその選定理由、各社で重点的に監視すべき項目を説明できる
- 8-K アラート（例：「Item 5.02 経営陣退任」）を見たとき、それがどれほど重要かを誰かに尋ねずに判断できる
- Earnings Call の議事録と 10-Q の主要データを読み、5分以内に自分の判断を形成できる
